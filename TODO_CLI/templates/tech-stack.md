# Tech Stack

把这里写成项目的技术边界，内容跟随项目类型生长——CLI 和 web app 需要写的肯定不同。

要覆盖的核心信息：

- **项目类型与入口**：是什么（CLI / web / API / 库……），怎么运行、交付到哪里
- **语言与运行时**：用什么语言、什么版本下限
- **核心选择**：关键的框架、库、存储方式，每一个都带上「选了谁 + 为什么选它」
- **测试与验证方式**：用什么测试框架、什么命令跑、什么算通过
- **约束**：明确哪些事不能做——不限只是「不得新增依赖」，还可以包括版本策略、数据安全、部署环境、兼容性要求等
- **已知空白**：暂时不做的、以后再说的事

格式自由。核心选择用表格、列表、段落都可以——目标是让人（或 AI）对照它就能判断实现是否踩线。
