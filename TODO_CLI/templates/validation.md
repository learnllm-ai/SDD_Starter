# Validation: <功能名>

把这里写成"怎么判断能合并"的清单，涵盖自动检查、场景验证和最终完成条件。

> 这里的每一步都由执行 feature 的一方（通常是 AI）来跑并核对结果，不是留给人去做的待办。

## 自动化检查

列出要跑的命令和期望结果。用表格、列表都可以。

## 场景验证

覆盖正常的、异常的和边界情况——逐条写清输入、步骤和期望输出，做到「另一人照着就能执行并判断通过还是失败」。常见的场景维度：

- 主流程（一条最小闭环先跑通）
- 空状态 / 首次使用
- 错误输入和失败路径
- 数据异常（如文件损坏）时的行为
- 环境兼容性（如干净环境下的首次运行）

## 输出文本检查

如果这个 feature 会输出对使用者可见的文本，检查语气、措辞是否和项目一致，没有占位或调试残留。

## 完成条件

满足以下所有条件才算「可以合并」——删掉不适用项，也可以追加项目特有的条件：

- 自动化检查全部通过
- 场景验证完成，无阻塞性问题
- 规格文件和实现一致（如有偏差已回写）
- roadmap 对应 phase 可被标记完成
- changelog 已更新（如项目启用）
